# TanStack Start app — AGENTS.md

Durable project context for humans and coding agents. Generated by scaffolding
with the TanStack CLI, then extending with the partner integrations below.

## Stack

- **Framework:** TanStack Start (React, file-router, SSR), TypeScript, Tailwind v4.
- **Package manager:** **bun** (bun.lock). NOT pnpm/npm, despite the `pnpm.*`
  block left in package.json by the CLI (harmless under bun).
- **Toolchain:** **eslint** (`@tanstack/eslint-config`, flat config) + Prettier.
- **Deployment:** **Cloudflare Workers** (`@cloudflare/vite-plugin`, `wrangler`).
- **Dev environment:** **Nix flake** (`flake.nix`) providing bun + nodejs_22 + git.

```sh
  --deployment cloudflare \
  --add-ons workos,sentry,prisma,neon \
  --no-install                     # deps installed later via bun inside the Nix shell
```

Deltas from the command in the original request: `pnpm → bun`, added
`--toolchain eslint`, added `--no-install`, used a scratch dir + merge.

```sh
npx @tanstack/intent@latest load @tanstack/start-client-core#start-core/server-functions
npx @tanstack/intent@latest load @tanstack/start-client-core#start-core/execution-model
```

Loaded skills that shaped this project:
- `server-functions`: `createServerFn({method}).validator(zod).handler(...)`,
  static imports, call fns with `({data})`, `useServerFn` only for redirect/notFound.
- `execution-model`: read `process.env` **inside the handler** (never module scope;
  on Workers env is per-request), server secrets must NOT use the `VITE_` prefix.

## Getting started

```sh
# 1. Enter the Nix dev shell (provides bun + node).
#    Flakes/nix-command are disabled on this machine, so pass the flag:
nix --extra-experimental-features 'nix-command flakes' develop
#    (To make it permanent, add to ~/.config/nix/nix.conf:
#     experimental-features = nix-command flakes)

# 2. Install deps.
bun install

# 3. Configure env: copy .env.example → .env.local and fill in real values.
#    At minimum set VITE_WORKOS_CLIENT_ID (the provider throws if it is empty —
#    a non-empty placeholder is shipped so the app boots).

# 4. Run it.
bun run dev          # http://localhost:3000 (or next free port)
bun run build        # outputs to dist/ (+ copies instrument.server.mjs)
bun run lint         # eslint (flat config)
bun run test         # vitest
```

## Environment variables (see `.env.example`)

`VITE_`-prefixed = bundled into the client (public). All others = server-only.

| Var | Scope | Used by |
| --- | --- | --- |
| `DATABASE_URL` | server | Neon raw client (`src/db.ts`), Prisma, `prisma generate` (must be non-empty) |
| `DATABASE_URL_POOLER` | server | Neon pooled connection (migrations) |
| `VITE_SENTRY_DSN` | client | Sentry init (`instrument.server.mjs`) |
| `VITE_SENTRY_ORG` / `VITE_SENTRY_PROJECT` | client | Sentry |
| `SENTRY_AUTH_TOKEN` | **server/CI** | Sentry source-map uploads |
| `VITE_WORKOS_CLIENT_ID` | client | WorkOS AuthKit — **provider throws if empty** |
| `VITE_WORKOS_API_HOSTNAME` | client | WorkOS (`api.workos.com`) |
| `SERPAPI_API_KEY` | **server** | `src/server/serpapi.ts` |
| `UNKEY_API_KEY` | **server (root key)** | `src/server/unkey.ts` |
| `UNKEY_API_ID` | server | `src/server/unkey.ts` (which API to verify against) |
| `VITE_ELECTRIC_SERVICE_URL` | client | Electric (future; not wired) |

`.env.local` ships with a **placeholder** `DATABASE_URL` (so `prisma generate`
resolves the var) and a **placeholder** `VITE_WORKOS_CLIENT_ID` (so the app
boots). Replace both with real values.

## Architectural decisions

- **bun, not pnpm.** The request said both; we confirmed bun and ship a Nix
  flake that provides it. The leftover `pnpm.*` key in package.json is inert.
- **eslint toolchain.** `--toolchain eslint` is a real CLI add-on; flat config
  in `eslint.config.js`. `no-unnecessary-condition` is turned **off for the
  generated `src/routes/demo/**`** files only (their runtime guards trip it);
  it stays active elsewhere.
- **db.ts exports a lazy Prisma client via a Proxy.** This module is imported
  from route files that also end up in the **client bundle**, so PrismaClient
  must not be constructed (or read env) at module scope — otherwise
  `new PrismaClient()` runs in the browser. First access happens inside a
  `createServerFn` handler (server-side).
- **Server secrets read `process.env` inside handlers**, never at module scope
  (Workers inject env per-request). SerpAPI and Unkey follow this.
- **`/grid` is `ssr: false`** (AG Grid is DOM-heavy), matching the WorkOS demo.
- **Build copies `instrument.server.mjs` to `dist/server`** (the Cloudflare
  build output), not `.output/server` (Node output, not produced here).

## Known gotchas

- **Network/proxy:** the host reaches the internet via a local proxy
  (`127.0.0.1:7897`). `registry.npmjs.org` is slow/drops there, so `.npmrc`
  pins **`registry.npmmirror.com`** (fast full mirror; also read by bun). To
  revert: delete `.npmrc`. `cache.nixos.org` and GitHub are reachable but slow;
  the nixpkgs flake input uses the **tarball URL** form (not `github:`) to avoid
  unauthenticated GitHub API rate-limiting (HTTP 403).
- **Nix flakes are disabled** on this machine → pass
  `--extra-experimental-features 'nix-command flakes'` to `nix develop`, or
  enable the feature in `~/.config/nix/nix.conf`. The flake pins
  `system = "x86_64-linux"` (`builtins.currentSystem` is unavailable in pure eval).
- **WorkOS provider throws at boot** if `VITE_WORKOS_CLIENT_ID` is empty — a
  non-empty placeholder is set in `.env.local` so the app starts.
- **`prisma generate` needs a non-empty `DATABASE_URL`** (the prisma config
  calls `env('DATABASE_URL')`); the placeholder satisfies it. Run `bun run
  db:generate` after changing the schema; the client outputs to
  `src/generated/prisma`.
- **`bun install` blocks 2 postinstalls by default** (`unrs-resolver`,
  `@sentry/cli`); both are trusted via `trustedDependencies` in package.json.
- **`serpapi` uses Node `http`** → relies on the `nodejs_compat` flag (already
  in `wrangler.jsonc`) on Workers. Local dev (Node) works without it.
- **miniflare (dev) prints `Request.cf` TimeoutError + "Falling back to a
  default placeholder"** — non-fatal; it is the Cloudflare dev runtime fetching
  geo data through the proxy and timing out. SSR still works.
- **`createServerFn().inputValidator()` is deprecated → use `.validator()`.**
  The CLI-generated demos (neon/prisma) still use `inputValidator`; this
  project's own server fns (`serpapi.ts`, `unkey.ts`) use the current
  `.validator(zod(...))`.

## Deployment (Cloudflare)

```sh
bun run deploy     # = bun run build && wrangler deploy
```

`wrangler.jsonc`: `main = "@tanstack/react-start/server-entry"`,
`compatibility_flags = ["nodejs_compat"]`. Secrets (e.g. `SERPAPI_API_KEY`,
`UNKEY_API_KEY`, `DATABASE_URL`) should be set with `wrangler secret put <NAME>`
or in the Cloudflare dashboard — never committed.

> **Sentry-on-Workers note:** `instrument.server.mjs` + the `NODE_OPTIONS`
> pattern is Node-oriented. For production Workers, confirm Sentry is initialized
> within the server entry (the add-on targets this, but verify on first deploy).
